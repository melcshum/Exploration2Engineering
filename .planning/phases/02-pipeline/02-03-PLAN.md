  <action>
The workflow at `.github/workflows/deploy.yml` requires two secrets to be configured in the GitHub repository before it can deploy:

1. **CLOUDFLARE_API_TOKEN** — A Cloudflare API token with permissions for Pages deployment
2. **CLOUDFLARE_ACCOUNT_ID** — The Cloudflare account ID that owns the Pages project

To create these secrets:

**Step 1: Get CLOUDFLARE_ACCOUNT_ID**
- Log in to Cloudflare Dashboard (dash.cloudflare.com)
- Copy your Account ID from the right sidebar of any Cloudflare Pages page or from Profile page

**Step 2: Create CLOUDFLARE_API_TOKEN**
- Go to Cloudflare Dashboard → Profile → API Tokens
- Click "Create Token" → Use "Edit Cloudflare Workers" template (it has Pages permissions)
- Or create a custom token with "Account Cloudflare Pages: Edit" permission
- Copy the generated token immediately (it will not be shown again)

**Step 3: Add secrets to GitHub repository**
- Go to your GitHub repository → Settings → Secrets and variables → Actions
- Click "New repository secret" for each:
  - Name: `CLOUDFLARE_API_TOKEN`, Value: paste the token from Step 2
  - Name: `CLOUDFLARE_ACCOUNT_ID`, Value: paste the account ID from Step 1
- Save both secrets

Per D-05: Both secrets must be configured as GitHub repository secrets. The workflow passes them directly to wrangler-action without using a wrangler.toml file (per D-06).
