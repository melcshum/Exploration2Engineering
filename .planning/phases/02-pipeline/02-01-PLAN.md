  <action>
Create the file `.github/workflows/deploy.yml` with the following content:

1. **Workflow name:** "Deploy to Cloudflare Pages"
2. **Triggers:**
   - `push` to `branches: [main]`
   - `pull_request` to `branches: [main]` (for CI validation only, no deploy)
3. **Permissions:** `contents: read`, `deployments: write`
4. **Job:** `deploy` on `ubuntu-latest`
5. **Steps:**
   a. Checkout: `actions/checkout@v6`
   b. Set up Python: `actions/setup-python@v5` with `python-version-file: book/.python-version`
   c. Install dependencies: `pip install -r book/requirements.txt` with `working-directory: book`
   d. Build MkDocs: `mkdocs build --strict` with `working-directory: book`
   e. Deploy to Cloudflare Pages: `cloudflare/wrangler-action@v3` with:
      - `apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}`
      - `accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}`
      - `preCommands: wrangler pages project create exploration-to-engineering || true`
      - `command: pages deploy site --project-name=exploration-to-engineering`

Key constraints per D-XX decisions:
- D-01: No deploy on PRs — only on push to main
- D-03: Use preCommands to provision project if absent (|| true handles existing project)
- D-04: Must use cloudflare/wrangler-action@v3 (NOT pages-action)
- D-06: No wrangler.toml — pass apiToken and accountId directly to action
- D-10: No notification on failure — let workflow fail naturally
- D-11: Build command is mkdocs build --strict

The working-directory for pip install and mkdocs build is `book` because mkdocs.yml is at book/mkdocs.yml and the build reads site_dir: ../site from that location.
  </action>
